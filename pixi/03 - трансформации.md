# Трансформации и координаты

> Каждый display object имеет локальную матрицу; мировые координаты = цепочка матриц от `stage` до объекта.

## Свойства transform

| Свойство | Эффект |
|----------|--------|
| `x`, `y` | Позиция относительно **родителя** |
| `scale.x`, `scale.y` | Масштаб (1 = 100%) |
| `rotation` | Радианы вокруг **pivot** |
| `skew` | Наклон |
| `width`, `height` | Для Sprite — меняют scale относительно текстуры |

Порядок применения (концептуально): scale → rotate → translate (реализация — единая матрица).

## pivot vs anchor

| | Где используется | Смысл |
|---|------------------|--------|
| **anchor** | `Sprite`, `Text` (0–1) | Точка привязки **текстуры** к позиции: (0.5, 0.5) — центр |
| **pivot** | Любой display object (пиксели) | Точка вращения/позиционирования в **локальных** координатах объекта |

```ts
sprite.anchor.set(0.5); // центр спрайта в x,y
container.pivot.set(100, 100); // вращение вокруг (100,100) локально
```

На собесе: **anchor** — доли от размера текстуры; **pivot** — пиксели от верхнего левого угла bounds.

## Local vs World

```ts
// Локальные координаты точки → мировые
const global = sprite.toGlobal({ x: 10, y: 20 });

// Мировые → локальные дочернего
const local = container.toLocal(global, undefined, undefined, sprite);
```

**`getBounds()`** — AABB в координатах родителя/скипа (учитывает детей). Для hit-test и culling.

## Родительские трансформации

Ребёнок наследует position/scale/rotation родителя. Камера — обычно `Container`, который двигают, а мир внутри него.

```ts
const camera = new Container();
app.stage.addChild(camera);
camera.addChild(world); // двигаем camera — двигается весь world
```

## alpha и visibility

- `alpha` — прозрачность (0–1), **наследуется** (умножается по цепочке).
- `worldAlpha` — итоговая после родителей.
- `visible: false` — поддерево не рисуется.

## Частые вопросы

**Почему спрайт «прыгает» при rotation?**  
Неверный anchor/pivot — точка вращения не в центре визуала.

**Как сделать follow камеру?**  
Двигать container-«камеру» или `stage.pivot` + позиция игрока в центре экрана.

**Разница position и x/y?**  
В v8 `position` — `ObservablePoint`, `x`/`y` — те же данные.

## Шпаргалка

> Координаты всегда относительно родителя. anchor — для спрайтов (0–1), pivot — точка вращения в px. toGlobal/toLocal — перевод между системами координат. Родительский scale влияет на детей и на hit areas.
