# Производительность

> Цель: меньше **draw calls**, меньше **пересоздания GPU-ресурсов**, стабильный **frame time**.

## Главные рычаги

### 1. Batching и атласы

- Один **spritesheet** на экран/сцену где возможно.
- Сортировать детей так, чтобы одна текстура шла подряд (если нет z-order конфликтов).
- Избегать чередования «текстура A, B, A, B».

### 2. Меньше объектов с особыми состояниями

- Без фильтров/масок на массовых объектах.
- `Graphics` — не для 5000 обновляемых линий каждый кадр.

### 3. Object pooling

```ts
const pool: Sprite[] = [];

function acquire(): Sprite {
  return pool.pop() ?? new Sprite(texture);
}

function release(s: Sprite) {
  s.visible = false;
  pool.push(s);
}
```

Пули для пуль, частиц, врагов — меньше alloc/GC JS.

### 4. Текст и UI

- `BitmapText` для часто меняющихся цифр.
- `Text` обновлять только при изменении строки.
- Статический UI — один container, `cacheAsTexture` (если подходит):

```ts
container.cacheAsTexture(true); // v8: rasterize subtree в текстуру
// при изменении детей — cacheAsTexture(false) и снова true
```

**Осторожно:** cache увеличивает VRAM; при частых изменениях детей — хуже.

### 5. Resolution

```ts
await app.init({
  resolution: Math.min(window.devicePixelRatio, 2),
});
```

На слабых устройствах — `resolution: 1` для прироста FPS.

### 6. Culling

Не обновлять/не показывать объекты далеко за кадром:

```ts
if (!viewBounds.intersects(sprite.getBounds())) {
  sprite.renderable = false;
}
```

### 7. Загрузка ассетов

- `Assets.backgroundLoad` для следующего уровня.
- Текстуры POT/разумного размера — VRAM.

### 8. Память GPU

- `destroy()` / `Assets.unload` → [10 - очистка памяти](./10%20-%20очистка%20памяти.md)
- Массовый destroy — порциями по кадрам.

## Метрики на собесе

| Метрика | Что значит |
|---------|------------|
| **FPS** | Кадры/с; 60 — цель на desktop |
| **Draw calls** | Чем меньше, тем лучше (сотни ок, тысячи — плохо) |
| **VRAM** | Текстуры + RT фильтров |
| **Frame time** | ms на кадр; 16.6ms ≈ 60 FPS |

Инструменты: Chrome Performance, Spector.js (WebGL), `renderer.texture.managedTextures` / debug плагины Pixi.

## Антипаттерны

- Новый `Sprite` каждый выстрел без pool.
- `blur` на десятках объектов.
- `console.log` в ticker.
- Загрузка больших PNG синхронно в gameplay.
- 4K текстуры для иконок 64×64.

## AnimatedSprite

```ts
import { AnimatedSprite } from 'pixi.js';

const anim = new AnimatedSprite(textures);
anim.animationSpeed = 0.1;
anim.play();
```

Кадры из **одного атласа** — один source, хороший batching. Смена только `texture` в рамках того же source.

## Частые вопросы

**ParticleContainer?**  
В старых версиях — быстрый путь для простых частиц. В v8 смотреть **ParticleContainer** / community solutions — уточнять актуальный API в доке версии.

**WebGPU быстрее WebGL?**  
Часто на современном железе — да, но зависит от сцены и драйвера; не серебряная пуля.

**Как найти узкое место?**  
Профайлер: отключать фильтры, уменьшать число спрайтов, смотреть draw calls.

## Шпаргалка

> Производительность Pixi = batching (атласы, порядок), минимум фильтров/масок/dynamic Graphics, pooling, разумный resolution, culling, своевременный destroy GPU. Измеряем draw calls и frame time, не только FPS.
