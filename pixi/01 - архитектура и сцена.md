# Архитектура и сцена (Scene Graph)

> PixiJS — 2D WebGL/WebGPU-рендерер. Всё видимое — дерево **display objects**, которое обходит рендерер каждый кадр.

## Ключевые сущности

| Класс | Роль |
|-------|------|
| `Application` | Точка входа: canvas, renderer, ticker, stage |
| `Renderer` | Рисует сцену (WebGL / WebGPU / Canvas fallback) |
| `Container` | Узел дерева, может иметь детей, сам по себе не рисуется пиксели* |
| `Sprite`, `Text`, `Graphics`… | Листья, которые реально попадают в draw calls |

\*Container участвует в трансформациях и clipping детей.

```ts
import { Application, Container, Sprite } from 'pixi.js';

const app = new Application();
await app.init({ width: 800, height: 600, background: '#1099bb' });
document.body.appendChild(app.canvas);

const root = new Container();
app.stage.addChild(root);

const hero = new Sprite(texture);
root.addChild(hero);
```

## Scene Graph (дерево сцены)

- **`app.stage`** — корневой `Container`, всё добавляется сюда или в его потомков.
- **Порядок детей** = порядок отрисовки (кто позже в `children`, тот «сверху»).
- **`addChild` / `removeChild`** — только связь parent ↔ child, память не освобождается без `destroy()`.
- **`visible: false`** — объект и потомки не рендерятся, но остаются в дереве.
- **`renderable: false`** — не рисуется, но может участвовать в bounds/логике (зависит от версии/настроек).

## Частые вопросы на собесе

**Чем `Application` отличается от `Renderer`?**  
`Application` — обёртка «под ключ»: создаёт renderer, stage, ticker, resize. `Renderer` можно использовать отдельно, если нужен свой game loop и DOM.

**Что такое display list?**  
Исторически — список объектов к отрисовке. В v8 это обход **scene graph**: рендерер рекурсивно обходит дерево, собирает renderables для batching.

**Container vs Sprite?**  
`Container` группирует и трансформирует детей. `Sprite` — квад с текстурой, лист batching.

**`zIndex` / сортировка?**  
У `Container` можно включить `sortableChildren = true` и задавать `child.zIndex` — дети пересортируются перед рендером.

**Сколько Application на странице?**  
Обычно один на canvas. Несколько — отдельные canvas и renderer’ы (тяжелее по GPU).

## Шпаргалка

> Pixi — scene graph: `Application` держит `stage`, в него вешаем `Container` и листья (`Sprite`, `Text`…). Порядок в `children` задаёт z-order. Рендерер обходит дерево и батчит draw calls. Удаление из дерева ≠ освобождение GPU — нужен `destroy()`.
