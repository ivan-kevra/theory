# Текстуры и ассеты

> **Texture** — «вид» на данные. **TextureSource** — источник пикселей (Image, Canvas, Video, Buffer). Один source → много Texture (разные frame/region).

## Цепочка (v8)

```
Image / Canvas / Video / Buffer
        ↓
   TextureSource  (GPU upload, размер, format)
        ↓
   Texture        (frame, UV, rotate)
        ↓
   Sprite(texture)
```

В v7 говорили `BaseTexture` + `Texture` — на собесе v8: **TextureSource** вместо BaseTexture.

## Загрузка через Assets

```ts
import { Assets, Sprite } from 'pixi.js';

await Assets.load([
  'hero.png',
  { alias: 'sprites', src: 'atlas.json' }, // spritesheet
]);

const hero = Sprite.from('hero.png');
const coin = Sprite.from('sprites', 'coin'); // frame из атласа
```

| API | Назначение |
|-----|------------|
| `Assets.load(url)` | Загрузка + кэш |
| `Assets.get(alias)` | Из кэша |
| `Assets.unload(alias)` | Выгрузка GPU + кэш |
| `Assets.backgroundLoad` | Предзагрузка в фоне |

## Spritesheet / атлас

- Один PNG + JSON (TexturePacker, etc.) → много `Texture` с общим **TextureSource**.
- **Меньше draw calls** — спрайты из одного атласа батчатся вместе.
- При смене кадра анимации меняют `sprite.texture = textures[i]`, source часто один.

## Размеры и степени двойки

- Старые GPU любили POT (power of two); современный WebGL часто ок с NPOT, но **mipmap/filter** могут ограничиваться.
- На собесе: «большие текстуры = VRAM; атласы экономят переключения текстур».

## UV, frame, trim

- `texture.frame` — прямоугольник в source.
- Trimmed spritesheet — прозрачные поля обрезаны, **anchor** учитывает смещение (см. metadata в JSON).

## Динамические текстуры

```ts
// Canvas как source
const canvas = document.createElement('canvas');
const texture = Texture.from(canvas);
// каждый кадр после рисования на canvas:
texture.source.update();
```

Video: `Texture.from(videoElement)`, обновление кадров по `update`.

## Частые вопросы

**Shared texture?**  
Несколько спрайтов с одной `Texture` — один GPU-ресурс. `destroy({ texture: true })` у одного спрайта убьёт текстуру для всех.

**Чем `Sprite.from('a.png')` отличается от `new Sprite(texture)`?**  
`from` берёт/создаёт текстуру из кэша Assets или синхронно из уже загруженного.

**Зачем атлас?**  
Batching + один HTTP + меньше переключений текстур.

## Шпаргалка

> TextureSource хранит пиксели на GPU, Texture — frame/UV для Sprite. Assets — загрузка и кэш. Атласы критичны для batching. Выгрузка: `Assets.unload` или `destroy`/`source.unload` — см. [10 - очистка памяти](./10%20-%20очистка%20памяти.md).
